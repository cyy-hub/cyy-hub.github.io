2.导航栏菜单定制化显示，主要原理：

  1. 按hexo 提供的文章归类功能（Categories），在文章Front-matter Categories写上相应的类别，在生成静态网页信息时，会在public/categories/下生成相应类别的文件夹，用于管理该类的文件。

  2. 在配置首页的menu信息时，将导航栏项目配置为相应的文件夹即可，

  3. 导航栏项目的中文名称：对应主题文件夹下的language/zh-CN.yml 提供中英文对照表。（改中英文对照表不起作用，直接写中文）

5. hexo 电脑迁移

1.新电脑下安装必须的环境：node,npm -> 安装hexo【 npm install -g hexo-cli】

2.原有博客目录拷贝至新电脑

3.配置和git hub 的pub key [多平台和多用配置]和user.name 、user.email[在.deploy_git 配置]

ssh-keygen -t rsa -f ~/.ssh/id_rsa.cyy # cyy 个人信息标识符号， 生成个人专属密钥

vim ~/.ssh/config # ssh 配置文件修改访问密钥文件位置， 指定Host， 访问用户以及对应的密钥文件

cat ~/.ssh/id_rsa.cyy.pub # 将公钥复制到GitHub 平台上

cd xx/Blog/.deploy_git # 为版本库设置特定user.name 、user.email，

git config user.name “cyy1” # 如果存在多个git 平台，建议不要使用global参数设置
git config user.email “cyy1@163.com”
